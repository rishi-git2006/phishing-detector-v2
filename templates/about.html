About PhishGuard

A machine-learning-powered phishing email detection system built with Python, Flask, and scikit-learn. Developed as a cybersecurity portfolio project demonstrating applied NLP and threat detection techniques.

Tech Stack

  • Python 3.10+ Core language
  • Flask Web framework & routing
  • scikit-learn ML models (LR, NB, RF) and TF-IDF
  • pandas / numpy Data handling and feature engineering
  • scipy Sparse matrix operations for combined features
  • joblib Model serialisation
  • Jinja2 HTML templating (auto-escaping enabled)

ML Pipeline

  1. Dataset — 200+ labelled emails (phishing + legitimate) covering diverse attack types: credential harvesting, prize scams, CEO fraud, delivery scams, and more.
  2. TF-IDF vectorisation — 8,000-feature 1–3-gram representation with sublinear TF scaling and English stop-word removal.
  3. Feature engineering — 18 linguistic signals (urgency, credentials, threats, spoofing) + 9 URL signals (typosquats, shorteners, suspicious TLDs).
  4. Training — Three candidate models trained: Multinomial Naive Bayes, Logistic Regression, Random Forest. Best selected by macro F1.
  5. Validation — Stratified 80/20 split + 5-fold cross-validation to detect overfitting and confirm generalisation.

Cybersecurity Features

  • 🔗 URL typosquat detection — Levenshtein-distance check against 50+ popular brands
  • 🌐 TLD abuse detection — Flags 40+ high-abuse TLDs (.tk, .xyz, .gq …)
  • 🔤 Character substitution detection — Catches paypa1, g00gle-style tricks
  • 👤 Sender spoofing detection — Identifies false authority claims ("This is PayPal")
  • ⚠️ Urgency & threat analysis — 60+ urgency and threat phrases
  • 💰 Social engineering bait — Prize/money/inheritance scam patterns
  • 🛡️ Input validation — Length limits, null-byte stripping, Jinja2 auto-escaping
  • 📊 Explainability — Every verdict is backed by specific, human-readable flags

Limitations & Future Work

  • Dataset is hand-crafted (~200 emails). Integrating real-world corpora (Enron, SpamAssassin) would significantly improve accuracy.
  • Model does not analyse email headers (From/Reply-To spoofing, DKIM/SPF status). Header analysis is a high-value future addition.
  • No live URL checking — URLs are analysed structurally, not fetched. WHOIS age and reputation lookups would reduce false negatives.
  • Multilingual phishing emails are not handled — model is trained on English text only.
  • A feedback loop (user-reported misclassifications feeding back into training) would enable continuous improvement in production.
